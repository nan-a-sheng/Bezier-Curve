# 计算机图形学实验三：交互式贝塞尔曲线绘制（Taichi + Python）

## 一、实验目标
1. 理解贝塞尔曲线（Bézier Curve）的几何意义与控制点作用。
2. 掌握并编码实现求取贝塞尔曲线的 De Casteljau 递归插值算法。
3. 理解光栅化、帧缓存（Frame Buffer）、像素寻址与颜色赋值底层原理。
4. 掌握图形界面鼠标点击添加控制点、键盘清空画布的交互事件处理。
5. 遵循现代高性能渲染规范：CPU批量计算 + GPU并行绘制，减少跨设备数据传输开销。

## 二、数学原理：De Casteljau 算法
贝塞尔曲线由一组控制点 \(P_0,P_1,\dots,P_{n-1}\) 定义，引入参数 \(t\in[0,1]\)，t 从 0 变化到 1 即可遍历整条曲线。

算法核心流程：
1. 给定一组原始控制点与参数 \(t\)；
2. 相邻两点做线性插值：  
\[P'_i=(1-t)\cdot P_i+t\cdot P_{i+1}\]
得到数量少 1 的新点集；
3. 对上一轮生成的新点集重复插值，不断降阶；
4. 最终仅剩一个点，即为当前 \(t\) 对应贝塞尔曲线上精确坐标。

该算法数值稳定性高，是工程上生成贝塞尔曲线的标准实现方式。

## 三、光栅化基础原理
1. 程序开辟 **800×800 像素缓冲区（Taichi Field）**，模拟显存帧缓存；
2. De Casteljau 输出为 \([0,1]\) 归一化浮点坐标；
3. 将浮点坐标 × 屏幕分辨率，映射为整型像素索引；
4. 对合法坐标（边界校验）直接修改像素 RGB 值，完成“点亮像素”光栅化操作。

🧠 算法原理（实验报告核心内容）

1. 反走样贝塞尔曲线

核心思路

传统光栅化将浮点坐标直接截断为整数，导致曲线边缘出现阶梯状锯齿（走样）。本项目采用「亚像素级权重混合」方案，通过3×3邻域采样与高斯衰减模型，实现曲线边缘的平滑过渡。

实现步骤（实验报告重点）

1. 采样：使用De Casteljau算法，对贝塞尔曲线进行高精度采样，得到浮点坐标(x, y)（避免直接截断导致的走样）。

2. 邻域选取：以浮点采样点为中心，选取3×3像素邻域，覆盖该点的亚像素影响范围。

3. 距离计算：计算每个像素中心点到浮点采样点的欧氏距离，量化像素对曲线的贡献度。

4. 权重分配：采用高斯衰减模型，距离越近，权重越高（公式：$$weight = e^{-dist^2 / 1.0}$$），实现自然平滑过渡。

5. 颜色混合：按权重混合背景色与曲线色，避免硬边缘，最终实现反走样效果。

2. 均匀三次B样条曲线

核心思路

贝塞尔曲线存在「全局控制」和「阶数与控制点数量绑定」的缺陷，均匀三次B样条通过「分段三次多项式+局部基函数」，解决上述问题，实现局部控制与天然平滑拼接。

数学基础（实验报告公式参考）

均匀三次B样条曲线的点计算公式为：

$$P(u) = \begin{bmatrix} u^3 & u^2 & u & 1 \end{bmatrix} M_B \begin{bmatrix} P_i \\ P_{i+1} \\ P_{i+2} \\ P_{i+3} \end{bmatrix}, \quad u \in [0,1]$$

其中，$$M_B$$ 为均匀三次B样条基矩阵（预定义，避免递归计算）：

$$M_B = \frac{1}{6} \begin{bmatrix} -1 & 3 & -3 & 1 \\ 3 & -6 & 3 & 0 \\ -3 & 0 & 3 & 0 \\ 1 & 4 & 1 & 0 \end{bmatrix}$$

实现逻辑

每4个相邻控制点生成一段三次曲线，n个控制点对应n-3段曲线；曲线分段连接处天然满足C²连续（二阶导数连续），视觉上无断点、无突兀感。

📝 实验结果分析（实验报告专用）

1. 反走样效果验证

实验结果表明，采用3×3邻域高斯权重混合的反走样方案，可有效消除贝塞尔曲线的阶梯状锯齿。对比传统光栅化（无反走样），反走样版本的曲线边缘过渡自然，视觉平滑度显著提升，符合计算机图形学中反走样技术的核心目标——减少采样误差导致的视觉失真。

2. 控制特性验证

- 贝塞尔曲线：拖动任意一个控制点，整条曲线都会发生变形，体现了「全局控制」的特性，适合简单形状的整体调整，但不适合复杂形状的局部修改。

- 均匀三次B样条曲线：拖动中间控制点，仅影响该点相邻的3段曲线，其他区域曲线保持不变，体现了「局部控制」的核心优势，解决了贝塞尔曲线的全局控制缺陷，可用于复杂曲线的精细建模。

3. 连续性验证

通过交互操作可观察到，均匀三次B样条曲线的分段连接处无断点、无突兀感，验证了其天然C²连续的特性；而贝塞尔曲线若进行分段拼接，需额外约束控制点位置才能实现连续，否则会出现明显的拼接痕迹。

4. 性能分析

程序运行帧率稳定在60FPS，贝塞尔曲线采样数设为400、B样条曲线采样数设为100时，既能保证曲线平滑度，又不会出现卡顿。反走样的3×3邻域计算量较小，对CPU性能要求较低，可满足实验演示的实时性需求。

1. 贝塞尔曲线核心函数

def bezier_point(points, t):
    # 功能：使用De Casteljau算法，计算贝塞尔曲线在参数t处的点（t∈[0,1]）
    # 参数：points-控制点列表，t-采样参数（0对应起点，1对应终点）
    # 返回：采样点的浮点坐标(x, y)
    p = [pygame.Vector2(x, y) for x, y in points]  # 转换为向量，便于计算
    n = len(p)
    for k in range(1, n):  # 迭代计算，每轮减少一个控制点
        for i in range(n - k):
            # 线性插值公式：(1-t)*前一点 + t*后一点
            p[i] = (1 - t) * p[i] + t * p[i+1]
    return (p[0].x, p[0].y)  # 返回最终采样点

def draw_antialiased_bezier(screen, points, color, samples=400):
    # 功能：绘制反走样贝塞尔曲线，核心实现反走样逻辑
    # 参数：screen-渲染窗口，points-控制点，color-曲线颜色，samples-采样数（越高越平滑）
    for i in range(samples + 1):
        t = i / samples  # 均匀采样t参数
        x, y = bezier_point(points, t)  # 得到高精度浮点采样点
        # 3×3邻域反走样：遍历采样点周围的9个像素
        for dx in (-1, 0, 1):
            for dy in (-1, 0, 1):
                cx = x + dx  # 邻域像素的x坐标
                cy = y + dy  # 邻域像素的y坐标
                # 计算像素中心点到采样点的欧氏距离
                dist = math.hypot(x - cx, y - cy)
                # 高斯衰减权重：距离越近，权重越高，实现平滑过渡
                weight = math.exp(-dist ** 2 / 1.0)
                # 按权重计算最终颜色，避免颜色溢出（0-255）
                r = min(255, int(color[0] * weight))
                g = min(255, int(color[1] * weight))
                b = min(255, int(color[2] * weight))
                # 边界检查：避免像素越界导致报错
                if 0 <= int(cx) < WIDTH and 0 <= int(cy) < HEIGHT:
                    screen.set_at((int(cx), int(cy)), (r, g, b))

2. 均匀三次B样条核心函数

def bspline_point(p0, p1, p2, p3, u):
    # 功能：计算单段均匀三次B样条曲线在参数u处的点（u∈[0,1]）
    # 参数：p0-p3-4个相邻控制点，u-采样参数
    # 原理：基于预定义基矩阵，通过多项式加权求和计算采样点
    u2 = u * u  # u的平方
    u3 = u2 * u  # u的立方
    # 三次B样条基函数系数（由基矩阵M_B计算得出，简化计算）
    c0 = (-u3 + 3*u2 - 3*u + 1) / 6
    c1 = (3*u3 - 6*u2 + 4) / 6
    c2 = (-3*u3 + 3*u2 + 3*u + 1) / 6
    c3 = u3 / 6
    # 控制点加权求和，得到采样点坐标
    x = p0[0]*c0 + p1[0]*c1 + p2[0]*c2 + p3[0]*c3
    y = p0[1]*c0 + p1[1]*c1 + p2[1]*c2 + p3[1]*c3
    return (x, y)

def draw_uniform_cubic_bspline(screen, ctrl_pts, color, samples=100):
    # 功能：绘制完整的均匀三次B样条曲线
    # 参数：ctrl_pts-所有控制点，samples-每段曲线的采样数
    n = len(ctrl_pts)
    if n < 4:  # 三次B样条至少需要4个控制点，否则不绘制
        return
    # 遍历所有分段（n个控制点对应n-3段曲线）
    for i in range(n - 3):
        p0 = ctrl_pts[i]
        p1 = ctrl_pts[i+1]
        p2 = ctrl_pts[i+2]
        p3 = ctrl_pts[i+3]
        # 每段曲线均匀采样，确保平滑
        for j in range(samples + 1):
            u = j / samples
            x, y = bspline_point(p0, p1, p2, p3, u)
            # 边界检查，避免越界
            if 0 <= int(x) < WIDTH and 0 <= int(y) < HEIGHT:
                pygame.draw.circle(screen, color, (int(x), int(y)), 1)

📝 实验总结与反思（实验报告参考）

本次实验通过Python+Pygame，成功实现了反走样贝塞尔曲线与均匀三次B样条曲线的可视化与交互，验证了两种曲线的几何特性与反走样技术的有效性，主要收获如下：

1. 反走样技术的核心是「亚像素级采样与权重混合」，通过3×3邻域高斯衰减，可有效消除光栅化导致的锯齿，提升曲线视觉质量。

2. 贝塞尔曲线的全局控制特性适合简单形状建模，但难以实现复杂形状的局部调整；均匀三次B样条的局部控制、固定阶数、天然C²连续等特性，更适合复杂曲线建模。

3. 交互式实现让抽象的曲线算法变得直观，通过拖拽控制点、切换曲线模式，加深了对De Casteljau算法、B样条基函数、分段拼接等核心知识点的理解。

4. 实验过程中遇到了语法错误、环境依赖、曲线不显示等问题，通过排查代码、调试环境，提升了代码调试与问题解决能力，也认识到算法实现中边界判断、参数设置的重要性。

后续可进一步优化：为B样条曲线添加反走样功能、支持控制点的添加/删除、实现曲线截图保存，进一步完善实验功能，提升演示效果。
